# 技能：write-article

根据微信公众号账号主题和指定热点撰写文章，输出的文件格式为html格式。

## 前置条件

- 已运行 `/wx.setup` 配置账号主题
- 已有待撰写的热点话题

## 执行流程

**禁止合并步骤执行**，先列出todos，执行完成标记完成。

- [ ] step 1: 确认写作需求
- [ ] step 2: 加载上下文
- [ ] step 3: 分析热点
- [ ] step 4: 确定切入角度
- [ ] step 5: 撰写文章
- [ ] step 6: 生成封面和摘要
- [ ] step 7: 文章审查
- [ ] step 8: 保存文章

### 1. 确认写作需求

询问用户：

- **文章主题/热点**: 用户要写的具体话题
- **文章类型**: 资讯类 / 观点类 / 干货类 / 故事类
- **目标字数**: 如1500-2000字
- **特殊要求**: 如有

### 2. 加载上下文

读取工作目录下的文件：

- `account_profile.md` - 账号主题配置
- `style_guide.md` - 文风与结构统一规范（最高优先级）
- `article_template.md` - 模板说明

**写作必须严格遵守 `style_guide.md` 中的规范**

### 3. 分析热点

- 搜索热点话题的相关内容
- 提取核心矛盾
- 提取用户评论关注点
- 提取情绪（焦虑 / 愤怒 / 好奇）

### 4. 确定切入角度

- 如何将热点、读者关注点与账号核心话题结合
- 目标读者关心什么
- 是否存在认知误区
- 是否可以制造冲突或反差

能否为用户提供**真正有价值**的内容？如果没有，只进行信息整理汇总，不发表任何观点。

### 5. 撰写文章（结构优先）

写作顺序必须为：

1. 按 style_guide.md 完成“纯内容结构”
2. 再映射为 HTML

禁止：

- 直接套HTML模板生成内容
- 忽略结构骨架

#### 模块使用策略

根据结构动态选择模块：

| 结构类型 | 必用模块            | 可选模块  |
| -------- | ------------------- | --------- |
| 冲突型   | key-point（反常识） | quote     |
| 干货型   | list（步骤）        | key-point |
| 故事型   | blockquote（情绪）  | summary   |

#### 模板使用规则

- `article_template.md` 仅作为“样式参考”
- 不作为“结构约束”
- 可以：
  - 删除 summary
  - 调整段落顺序
  - 重复使用 key-point

**HTML 标签使用规范**：

- `h2` - 章节标题
- `p` - 段落
- `blockquote` - 金句 / 引用 / 情绪
- `section.key-point` - 重点/反常识
- `ul/ol` - 列表
- `strong` - 关键词强调
- `span.highlight` - 高亮

### 6. 生成封面和摘要

根据文章内容，生成：

- 封面图关键词描述（写入 HTML 注释）
- 文章摘要（用于微信预览）

### 7. 文章审查

1. 审查文章信息是否真实、准确，存在虚假信息或错误，及时修正。

2. 评分维度（1-5分）：

- Hook吸引力
- 反转强度
- 内容价值
- 情绪共鸣
- 可转发性

总分 < 15 分 → 自动重写优化

### 8. 保存文章

- 格式为 HTML 文档
- 保存路径：工作目录下的 `articles/article_YYYYMMDD_{话题关键词}.html`

## 错误处理

- 账号配置不存在：提示先运行 `/wx.setup`
- 热点搜索失败：提示用户检查网络连接
- 缺少模板文件：使用默认 HTML 结构和样式
- 写作失败：保存草稿，提示用户手动完成

# 技能：write-article

根据微信公众号账号主题和指定热点撰写 HTML 格式文章。

## 前置条件

- 已运行 `/wx.setup` 配置账号主题
- 已有待撰写的热点话题

## 执行流程

**禁止合并步骤执行**，先列出todos，执行完成标记完成。

### 1. 确认写作需求

询问用户：

- **文章主题/热点**: 用户要写的具体话题
- **文章类型**: 资讯类 / 观点类 / 干货类 / 故事类
- **目标字数**: 如1500-2000字
- **特殊要求**: 如有

### 2. 加载上下文

读取工作目录下的文件：

- `account_profile.md` - 账号主题配置
- `style_guide.md` - 文风指南
- `article_template.md` - HTML 文章模板（包含完整结构和样式）

### 3. 分析热点

- 搜索热点话题的相关内容
- 提取核心矛盾
- 提取用户评论关注点
- 提取情绪（焦虑 / 愤怒 / 好奇）

### 4. 确定切入角度

- 如何将热点、读者关注点与账号核心话题结合
- 目标读者关心什么
- 是否存在认知误区
- 是否可以制造冲突或反差
- 如果无法确定切入角度，可以不考虑账号定位，从普世价值切入

### 4.5 确定文章结构

在写作前，必须选择文章结构类型：

可选结构：

- 【冲突型】（优先级最高）
  - 适用：热点有争议 / 认知反差 / 可打脸
  - 结构：
    1. 反常识开头（打脸认知）
    2. 案例或现象
    3. 原因拆解
    4. 上升总结

- 【干货型】
  - 适用：知识、经验、方法论
  - 结构：
    1. 提出问题
    2. 提前给结论
    3. 分点解释（2-4点）
    4. 总结强化

- 【故事型】
  - 适用：情绪、案例、人物
  - 结构：
    1. 场景/人物
    2. 冲突
    3. 转折
    4. 结果 + 方法提炼

⚠️ 必须选择一种结构，不可混用
⚠️ 后续写作必须严格遵循该结构

### 5. 撰写 HTML 文章（结构优先）

先根据【步骤4.5选择的结构】完成内容写作，再映射到 HTML 模板。

要求：

- 内容结构必须优先于 HTML 模板
- 允许对模板结构进行调整（可删除/新增模块）
- 不强制使用所有模块（如 summary、quote、key-point）

#### 模块使用策略

根据结构动态选择模块：

| 结构类型 | 必用模块            | 可选模块  |
| -------- | ------------------- | --------- |
| 冲突型   | key-point（反常识） | quote     |
| 干货型   | list（步骤）        | key-point |
| 故事型   | blockquote（情绪）  | summary   |

#### 模板使用规则

- `article_template.md` 仅作为“样式参考”
- 不作为“结构约束”
- 可以：
  - 删除 summary
  - 调整段落顺序
  - 重复使用 key-point

**HTML 标签使用规范**：

- `h2` - 章节标题
- `p` - 段落
- `blockquote` - 金句 / 引用 / 情绪
- `section.key-point` - 重点/反常识
- `ul/ol` - 列表
- `strong` - 关键词强调
- `span.highlight` - 高亮

### 6. 生成封面和摘要

根据文章内容，生成：

- 封面图关键词描述（写入 HTML 注释）
- 文章摘要（用于微信预览）

### 7. 文章审查

- 标题吸引力
- 内容价值
- 是否有新意（避免模板感）

### 8. 保存文章

保存到工作目录下的 `articles/article_YYYYMMDD_{话题关键词}.html`

## 错误处理

- 账号配置不存在：提示先运行 `/wx.setup`
- 热点搜索失败：提示用户检查网络连接
- 缺少模板文件：使用默认 HTML 结构和样式
- 写作失败：保存草稿，提示用户手动完成

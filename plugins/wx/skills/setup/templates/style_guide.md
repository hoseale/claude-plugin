# 微信公众号文风指南

## 一、核心原则

### 1. 读者价值优先

每一段必须提供至少一种价值：

- 信息（知识/数据）
- 认知（观点/拆解）
- 情绪（共鸣/冲击）

无价值内容必须删除

### 2. 对话感表达

必须使用：

- “你”替代“用户/大家”
- 常用表达：
  - 但问题来了
  - 说白了
  - 本质是

### 3. 简洁表达

- 单句 ≤ 20字
- 单段 ≤ 4行
- 一句话只表达一个意思
- 禁止废话开头：
  - 随着时代发展
  - 众所周知

### 4. 刺激点规则

每200字必须包含至少1个：

- 反转
- 金句
- 数据
- 极端案例

## 二、统一内容结构

所有文章必须遵循以下底层结构：

### 1. 开头 Hook（必须）

- ≤ 3行
- 必须包含：
  - 反常识 / 冲突 / 利益点

示例：

你以为XXX  
但真相是XXX

---

### 2. 事件/问题引入

- 描述具体案例 / 现象
- 禁止纯抽象

### 3. 关键反转

必须出现：

但真正的情况是——

用于打破原认知

### 4. 本质拆解

说明：

- 为什么会这样
- 至少包含：
  - 数据 / 逻辑 / 类比

### 5. 方法/路径

适用于干货/观点内容：

- 至少2-4条
- 必须可执行

---

### 6. 结尾

必须包含：

- 1句总结性金句
- 1句转发引导

## 三、结构适配规则

在统一骨架基础上，允许风格偏移：

### 1. 冲突型（优先）

适用：

- 热点争议 / 打脸 / 认知差

强化：

- Hook更强
- 反转更狠

---

### 2. 干货型

适用：

- 方法 / 教程 / 经验

强化：

- 方法部分（第5步）

---

### 3. 故事型

适用：

- 人物 / 情绪 / 案例

强化：

- 第2步（场景）
- 第3步（冲突）

---

⚠️ 注意：

- 所有类型必须保留：
  - Hook
  - 反转
  - 金句

---

## 四、结构使用原则

统一结构为“隐形骨架”，禁止机械套用。

允许：

- 调整顺序
- 强化某一部分
- 弱化或省略某一部分

但必须保证：

- 有吸引（Hook）
- 有认知变化（反转）
- 有输出（结论/方法）

## 五、语言规则

必须：

- 多短句
- 多断句
- 多动词

示例：

❌ 提升内容生产效率  
✅ 原本1小时，现在10分钟

---

## 六、金句标准

好金句必须满足：

- 可截图
- 可传播
- 有观点

示例：

AI是工具，不是印钞机

---

## 七、标题原则

标题必须满足：

- 至少包含：
  - 反转 / 利益 / 好奇

示例：

- 你以为XXX，其实XXX
- 我研究了3天，发现XXX

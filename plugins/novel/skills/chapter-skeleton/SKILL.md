# 概述

基于分卷细纲，生成完整章节序列骨架，并完成：

- 章节预算
- 四幕结构
- 事件聚类
- 爽点分配

⚠️ 本技能只生成骨架，不展开章节细节

---

# 执行流程

## Step1：章节预算分配

**输入**：

- 卷细纲`03_Outline/volume/volume_[卷号].md`
- 世界观设定
- 主线大纲
- 角色设定

**计算**：

- 按3000–4000字/章
- 总章节数

**比例切分（四幕结构）**：

| 阶段 | 占比 | 内容定位       |
| ---- | ---- | -------------- |
| 起   | 15%  | 快速抓住读者   |
| 承   | 50%  | 连续爽点       |
| 转   | 20%  | 压制           |
| 合   | 15%  | 爆发、引出下卷 |

---

## Step2：事件聚类

将分卷细纲拆为“事件组”：

要求：

- 每幕 2–4 个事件组
- 每个事件组一句话描述
- 明确冲突与目标

---

## Step3：爽点分配

爽点类型：

- 打脸（身份反转）
- 逆袭（弱 → 强）
- 装逼（实力碾压）
- 反杀（绝境翻盘）
- 情绪爆发（复仇/守护）
- 智斗（策略取胜）

爽点分布规则：

- 每3章必须有爽点
- 不允许连续5章无进展

---

## Step4：章节骨架生成

每章必须明确是否包含爽点，不允许模糊描述

输出格式：

【第 X 卷：XXXX】

总章节：第1章 - 第XX章

一、起（第1-XX章）

第x章：[标题]
核心事件：
爽点：（如无写“无”）
事件组：
阶段标签：（铺垫/发展/反转/高潮）
...

二、承（第XX-XX章）

第x章：[标题]
核心事件：
爽点：（如无写“无”）
阶段标签：（铺垫/发展/反转/高潮）
...

三、转（第XX-XX章）

...

四、合（第XX-XX章）

...

---

# 输出位置

输出章节骨架到工作目录下`03_Outline/chapter/[卷号]_structure.md`

# 审核流程

检查以下项目：

- 是否符合四幕结构
- 是否存在节奏断裂
- 是否连续3章以上无爽点
- 是否偏离或冲突主线大纲`main_outline.md`
- 是否偏离或冲突分卷细纲`03_Outline/volume/volume_[卷号].md`
- 是否偏离或冲突世界观设定
- 是否偏离或冲突角色设定
- 时间线是否合理
- 战力 / 规则是否冲突
- 是否存在人设OOC

---

# 强约束

1. 每章必须有推进
2. 禁止重复事件
3. 禁止无目标章节

# 角色卡补充

如果存在新增主要角色，在工作目录`02_Characters`下补充角色卡

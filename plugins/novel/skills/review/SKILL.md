# 技能：review-logic

严苛的文学编辑和逻辑审查员。

## 执行流程

禁止合并步骤执行，先列出执行步骤列表，执行完成标记完成

- [ ] step1: 确定审查章节范围
- [ ] step2: 审查项
  - **字数要求**：最少3000中文字，3000-4000为最佳区间
  - **世界观冲突排查**：是否出现了设定中不存在的元素？
  - **大纲偏离排查**：本章内容是否偏离主线大纲方向？
  - **细纲偏离排查**：本章情节是否违背细纲中的具体规划？
  - **时间线矛盾排查**：人物行踪是否有分身术？时间流逝是否合理？
  - **战力/规则冲突排查**：是否出现战力崩坏或与设定矛盾的能力？
  - **人设OOC排查**：角色的决策是否违背了其核心动机？
  - **格式规范排查**：对话是否使用""、是否使用了禁止的转折句式、是否在末尾总结
- [ ] step3: 生成一份详细的《逻辑审查报告》，罗列潜在的问题，并给出修改建议。

## 错误处理

- 如果指定章节不存在，通知用户并询问调整范围。
- 如果世界观或角色文件缺失，提示用户先完善设定。
- 审查过程中发现严重矛盾时，暂停并征求用户处理意见。

## 分卷模式特有处理

- 如果分卷模式下的卷目录不存在，提示用户检查分卷配置
- 处理跨卷审查时，注意卷间的一致性和衔接
- 如果章节文件路径不符合分卷结构，尝试自动定位

# 技能：review-logic

严苛的文学编辑和逻辑审查员。

## 执行流程

禁止合并步骤执行，先列出执行步骤列表，执行完成标记完成

- [ ] step1: 确定审查章节范围
- [ ] step2: 验证项目结构
- [ ] step3: 读取基础数据
  - 读取世界观设定
  - 读取主线大纲
  - 读取人物卡
  - 读取风格指南
  - 读取章节概要汇总
  - 读取审查章节和上一章节
- [ ] step4: 进行以下维度的交叉比对：
  - **时间线排查**：人物行踪是否有分身术？时间流逝是否合理？
  - **战力/规则排查**：是否使用了世界观中不存在的魔法/科技？是否出现战力崩坏？
  - **人设OOC排查**：角色的决策是否违背了其核心动机？
  - **写作规则排查**：是否符合写作规则？
- [ ] step5: 生成一份详细的《逻辑审查报告》，罗列潜在的 Bug，并给出修改建议。

## 错误处理

- 如果指定章节不存在，通知用户并询问调整范围。
- 如果世界观或角色文件缺失，提示用户先完善设定。
- 审查过程中发现严重矛盾时，暂停并征求用户处理意见。

## 分卷模式特有处理

- 如果分卷模式下的卷目录不存在，提示用户检查分卷配置
- 处理跨卷审查时，注意卷间的一致性和衔接
- 如果章节文件路径不符合分卷结构，尝试自动定位

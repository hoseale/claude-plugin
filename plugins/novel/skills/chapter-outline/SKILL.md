# 概述

在不改变指定卷的章节骨架的前提下，逐批细化章节内容：

- 每次生成 3–5章或者指定章节范围
- 强化爽点、钩子与节奏

# 输入要求（必须提供）

1. 指定卷号
2. 对应卷号的章节骨架`03_Outline/chapter/[卷号]_structure.md`
3. 指定的章节范围（如第1-5章）
4. 已完成章节摘要（如有）`03_Outline/chapter/[卷号]_volume.md`

---

# 执行流程

## Step1：范围定位

从章节骨架中提取：

- 当前章节
- 所属阶段（起/承/转/合）
- 所属事件组

---

## Step2：上下文对齐

结合：

- 世界观、角色设定、主线大纲、分卷大纲
- 已完成章节摘要
- 当前章节目标

确保：

- 不重复
- 不断层
- 不跳剧情

---

## Step3：逐章设计

第X章：

- 事件组：[事件组名称]
- 核心事件：
- 功能标签：（爽点/铺垫/反转/伏笔/虐点）
- 爽点类型：（如有）
- 信息推进：（世界/人物/主线）
- 钩子：（读完整章的动力）
- 断章：(合适的断章位置, 让读者想读下一章)

---

### 黄金三章规则（仅前3章强制）

- 第1章：冲突出现 + 世界异常 + 钩子：重大疑问
- 第2章：金手指/外挂/异常强化
- 第3章：必须出现爽点

---

## Step4：断章与钩子强化

钩子类型：

- 信息差（他不知道真相）
- 危机（马上要死）
- 反转（其实是陷阱）
- 新目标（更大事件出现）

断章规则：

- 80%章节断在未完成事件
- 禁止平稳结尾

---

## Step5：节奏校验

| 校验项   | 检查标准                         |
| -------- | -------------------------------- |
| 爽点密度 | 检查是否连续3章以上没有爽点/进展 |
| 信息推进 | 每章必须推进                     |
| 钩子强度 | 读完整章的动力                   |
| 断章方式 | 读者想读下一章                   |

---

# 强约束

1. 禁止修改章节骨架
2. 禁止新增章节
3. 禁止跳过章节
4. 必须严格按顺序生成

# 输出位置

工作目录下：`03_Outline/chapter/[卷号]_volume.md`

# 审核流程

检查：

- 输出结果是否符合**执行流程**
- 世界观是否偏离或存在冲突
- 大纲`03_Outline/main_outline.md`是否偏离或存在冲突
- 卷纲`03_Outline/volume/volume_[卷号].md`是否偏离或存在冲突
- 章节骨架`03_Outline/chapter/[卷号]_structure.md`是否偏离或存在冲突
- 角色设定是否偏离或存在冲突
- 时间线矛盾排查：人物行踪是否有分身术？时间流逝是否合理？
- 战力/规则冲突排查：是否出现战力崩坏或与设定矛盾的能力？
- 人设OOC排查：角色的决策是否违背了其核心动机？

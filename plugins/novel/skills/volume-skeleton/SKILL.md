# 概述

构建全书级分卷结构体系，明确：

- 每一卷的地图边界
- 主角跨卷成长路径
- 主线冲突分阶段推进
- 各卷爽点节奏与定位

⚠️ 本技能不展开具体剧情，仅定义结构

---

# 执行流程

**禁止合并步骤执行**，先列出todos，执行完成标记完成。

## step0：输入内容

- 世界观设定
- 主线大纲
- 角色卡

## Step1：全书边界定义

- 总卷数
- 世界地图层级划分（如：下界 → 中域 → 上界 → 核心区域）

## Step2：主角成长总路径

| 卷数 | 实力 | 身份 | 资源 | 核心诉求 |
| ---- | ---- | ---- | ---- | -------- |

要求：

- 每卷必须有“阶段跃迁”
- 禁止停滞或跳跃失衡

## Step3：主线冲突分层推进

定义：

- 终极目标（最终对抗）
- 分阶段推进路径

| 卷数 | 核心矛盾 | 推进程度 | 是否接触终极反派 |
| ---- | -------- | -------- | ---------------- |

---

## Step4：地图体系拆分

| 卷数 | 主地图 | 入场方式 | 离场方式 | 地图层级 |
| ---- | ------ | -------- | -------- | -------- |

规则：

- 地图逐级升级
- 禁止重复横跳
- 每卷地图相对独立

---

## Step5：各卷定位与爽点策略

爽点类型：

- 打脸（身份反转）
- 逆袭（弱 → 强）
- 装逼（实力碾压）
- 反杀（绝境翻盘）
- 情绪爆发（复仇/守护）
- 智斗（策略取胜）

每一卷可以有多个爽点，每个爽点都有其主策略。

输出格式：

# 强约束

1. 必须一次性输出所有卷 ❗
2. 禁止展开具体剧情
3. 必须保证跨卷一致性
4. 每卷必须闭环 + 推进主线

# 审核流程

检查：

- 世界观是否偏离或存在冲突
- 主线大纲`main_outline.md`是否偏离或存在冲突
- 地图是否冲突
- 成长是否断层
- 主线是否递进
- 爽点是否疲劳
- 是否存在重复卷

存在问题时，及时修复

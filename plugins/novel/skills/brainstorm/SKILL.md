# 小说大纲生成器

逐步生成小说大纲，按模块创建，每个模块完成后可调整。

## 执行流程

禁止合并步骤执行，先列出todos，执行完成标记完成

- [ ] Step1: 确定基本信息
- [ ] Step2: 确定核心剧情/故事梗概
- [ ] Step3: 确定故事背景
- [ ] Step4: 主要角色设定
- [ ] Step5: 确定情节结构
- [ ] Step6: 确定章节大纲
- [ ] Step7: 结局设计
- [ ] Step8: 文风与语感定制
- [ ] Step9: 输出文档

### 1. 确定基本信息

- **作品标题**
- **作品类型**：言情/玄幻/悬疑等
- **目标读者**
- **预计篇幅**：短/中/长/超长篇
- **风格基调**：轻松/沉重/黑暗等

### 2. 确定核心剧情/故事梗概

**先定故事梗概，再设计角色**

- **一句话概括**（Hook）：[主角]+[困境]+[目标]+[阻碍]
  示例："便利店店员获得时间倒流能力，每次重启中找出杀死自己的真凶"

- **三句话概括**：
  1. 开场（主角处境）
  2. 核心事件（打破平静）
  3. 主要冲突（挑战）

- **核心主线**：起因、发展、高潮、结局

- **核心问题**：主角困境与读者悬念

- **主题立意**
  1. **核心主题**：中心思想
  2. **副主题**：辅助思想
  3. **主题深度**：浅层/深层
  4. **情感基调**：希望/绝望/成长等

### 3. 确定故事背景

- **时代设定**：现代/古代/未来/架空
- **地理环境**：主要地点
- **世界观**：社会结构、规则、文化
- **特殊设定**：魔法/科技/异能等

### 4. 主要角色设定

**角色为故事服务**

#### 主角

- **姓名**
- **年龄**
- **外貌**
- **性格**
- **背景**
- **核心动机**
- **成长弧线**

#### 重要配角

- **姓名、年龄、性格、关系**
- **作用**（推动/帮助/阻碍）

#### 反派/对立面

- **动机目标**
- **冲突点**
- **对立原因**

### 5. 确定情节结构

1. 推荐2-3种合适的情节结构模型（如英雄之旅、三幕剧等），简述理由。
2. 等待用户选择结构。
3. 按选定结构进行。

### 6. 确定章节大纲

1. 确定是否采用分卷模式（是/否）。
2. 确定大纲及细纲

- 单章字数要求3000～4000字，请合理规划
- **分卷模式**
  - 确定分卷大纲， 模板参考工作目录 `./volume_structure.md`
  - 确定每一卷的细纲， 模板参考工作目录下 `./volume_detailed.md`
- **不分卷模式**
  - 确定章节细纲， 模板参考工作目录下的 `./chapter_outline.md`

### 7. 结局设计

- **结局类型**：圆满/悲剧/开放/反转
- **伏笔回收**：之前埋下的伏笔如何回收
- **留白与想象**：给读者的思考空间

### 8. 文风与语感定制

1. 根据题材为写手制定《文风指南》
2. 提炼题材专属**名词库**
3. 提炼**禁用词汇**
4. 规则插入到`style_guide.md`最后

### 9. 输出文档

- `01_World/world_setting.md`：世界观
- `02_Characters/`：角色卡（每个角色单独文件）
- `03_Outline/main_outline.md`：主线大纲
- **分卷模式**
  - `03_Outline/volume_structure.md`：分卷大纲
  - `03_Outline/volume_<卷号>_detailed.md`：各卷的细纲模板（卷级细纲）
- **不分卷模式**
  - `03_Outline/chapter_outline.md`：章节大纲

## 版本控制

git commit -m "{概要}"

## 交互原则

1. **渐进式生成**：逐步输出
2. **实时调整**：随时修改
3. **智能推荐**：关键点提供建议
4. **灵活适配**：根据反馈调整
5. **保持连贯**：关联内容同步更新

## 注意事项

1. 信息不完整时用合理假设推进
2. 类型小说增加专业要素
3. 注意长短篇侧重点
4. 角色动机合理，避免工具人
5. 情节有起伏，不平铺直叙

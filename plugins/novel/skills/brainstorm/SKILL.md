# 小说大纲生成器

## 概述

逐步生成小说大纲，按模块创建，每个模块完成后可调整。

## 小说大纲核心内容

### 1. 基本信息

- **作品标题**
- **作品类型**：言情/玄幻/悬疑等
- **目标读者**
- **预计篇幅**：短/中/长/超长篇
- **风格基调**：轻松/沉重/黑暗等

### 2. 核心剧情/故事梗概

**先定故事梗概，再设计角色**

- **一句话概括**（Hook）：[主角]+[困境]+[目标]+[阻碍]
  示例："便利店店员获得时间倒流能力，每次重启中找出杀死自己的真凶"

- **三句话概括**：
  1. 开场（主角处境）
  2. 核心事件（打破平静）
  3. 主要冲突（挑战）

- **核心主线**：起因、发展、高潮、结局

- **核心问题**：主角困境与读者悬念

### 3. 主题立意

- **核心主题**：中心思想
- **副主题**：辅助思想
- **主题深度**：浅层/深层
- **情感基调**：希望/绝望/成长等

### 4. 故事背景

- **时代设定**：现代/古代/未来/架空
- **地理环境**：主要地点
- **世界观**：社会结构、规则、文化
- **特殊设定**：魔法/科技/异能等

### 5. 主要角色设定

**角色为故事服务**

#### 主角

- **姓名**
- **年龄**
- **外貌**
- **性格**
- **背景**
- **核心动机**
- **成长弧线**

#### 重要配角

- **姓名、年龄、性格、关系**
- **作用**（推动/帮助/阻碍）

#### 反派/对立面

- **动机目标**
- **冲突点**
- **对立原因**

### 6. 情节结构

1. 推荐2-3种合适的情节结构模型（如英雄之旅、三幕剧等），简述理由。
2. 等待用户选择结构。
3. 按选定结构进行。

### 7. 章节大纲

- **分卷规划**：根据篇幅判断是否分卷
- **分卷概要**：各卷概要
- **每章概要**
- **章节节奏**：快/慢交替

### 8. 结局设计

- **结局类型**：圆满/悲剧/开放/反转
- **伏笔回收**：之前埋下的伏笔如何回收
- **留白与想象**：给读者的思考空间

## 文风与语感定制

1. 根据题材为写手制定《文风指南》
2. 提炼题材专属**名词库**
3. 提炼**禁用词汇**
4. 规则插入到`style_guide.md`最后

## 使用流程

1. 询问小说类型、灵感、篇幅、风格等基本信息。
2. 按**小说大纲核心内容**逐步输出，每步完成后可调整。
3. 文风与语感定制
4. 版本控制：重要的构思节点自动提交git记录

## 输出格式

整理到项目目录

- `01_World/world_setting.md`：世界观
- `02_Characters/`：角色卡（每个角色单独文件）
- `03_Outline/main_outline.md`：全书总纲，如果采用分卷规划，需要包含分卷结构配置，模板参考项目下的 `./volume_structure.md`，没有则参考插件内置的`/templates/volume_structure.md`
- **分卷细纲规划输出**（如果进行了分卷规划）：
  - `03_Outline/volume_<卷号>_detailed.md`：各卷的细纲模板（卷级细纲），模板参考项目下的 `./volume_detailed.md`，没有则参考插件内置的`/templates/volume_detailed.md`

## 交互原则

1. **渐进式生成**：逐步输出
2. **实时调整**：随时修改
3. **智能推荐**：关键点提供建议
4. **灵活适配**：根据反馈调整
5. **保持连贯**：关联内容同步更新

## 注意事项

1. 信息不完整时用合理假设推进
2. 类型小说增加专业要素
3. 注意长短篇侧重点
4. 角色动机合理，避免工具人
5. 情节有起伏，不平铺直叙

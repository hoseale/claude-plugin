# 小说大纲生成器

## 概述

这是一个逐步生成小说大纲的工具。根据用户输入的信息，按模块逐步创建完整的小说大纲。每个模块完成后，用户可以随时调整修改。

## 小说大纲核心内容

### 1. 基本信息

- **作品标题**：小说的名称
- **作品类型**：言情/玄幻/悬疑/科幻/都市/历史/武侠/恐怖/儿童等
- **目标读者**：受众群体
- **预计篇幅**：短篇/中篇/长篇/超长篇
- **风格基调**：轻松/沉重/黑暗/温暖/热血/治愈等

### 2. 核心剧情/故事梗概 ✨

**【重要】先确定故事大概讲什么，再设计角色**

- **一句话概括**（Hook）：用一句话吸引读者，概括故事精华
- 公式：[主角] + [困境] + [目标] + [阻碍]
- 示例："一个普通的便利店店员意外获得时间倒流能力，必须在每次重启中找出杀死自己的真凶"

- **三句话概括**：详细版故事主线
- 第一句：故事的开场（主角是谁，在什么处境）
- 第二句：核心事件（发生了什么打破平静）
- 第三句：主要冲突（主角要面对什么挑战）

- **核心主线**：故事的核心事件链
- 故事起因：什么事件打破了现状？
- 故事发展：主角经历了什么？
- 故事高潮：最激烈的冲突是什么？
- 故事结局：最终走向何方？

- **核心问题**：故事要回答的关键问题
- 主角面临的核心困境是什么？
- 读者会好奇什么？（悬念设置）

### 3. 主题立意

- **核心主题**：小说要表达的中心思想
- **副主题**：辅助表达的思想
- **主题深度**：浅层娱乐/深层思考
- **情感基调**：希望/绝望/成长/救赎等

### 4. 故事背景

- **时代设定**：现代/古代/未来/架空
- **地理环境**：故事发生的主要地点
- **世界观**：社会结构、规则、文化
- **特殊设定**：魔法、科技、异能等超自然元素

### 5. 主要角色设定

**【基于故事梗概】角色的存在是为故事服务的**

#### 主角

- **姓名**
- **年龄**
- **外貌特征**
- **性格特点**
- **背景故事**
- **核心动机**（为什么他要追求目标？）
- **成长弧线**（从开始到结束的转变）

#### 重要配角

- **姓名、年龄、性格、与主角关系**
- **在故事中的作用**（推动剧情/提供帮助/制造障碍）

#### 反派/对立面

- **动机与目标**
- **与主角的冲突点**
- **为什么是主角的对立面？**

### 6. 情节结构

1. 首先，根据前面的题材，向我推荐 2-3 种最合适的情节结构模型（如：英雄之旅、三幕剧结构、救猫咪15节拍、网文升级流地图模式、快穿单元剧模式、无限流副本模式、悬疑剥洋葱模式等），并简述推荐理由。
2. 等待我选择或确认其中一种结构。
3. 严格按照我选定的结构模型，进行下一步。

### 7. 章节大纲

- **分卷规划**：根据小说篇幅，判断是否分卷
- **分卷概要**：如果分卷，每个卷的概要
- **每章概要**
- **章节节奏**：快节奏/慢节奏交替

### 8. 结局设计

- **结局类型**：圆满/悲剧/开放/反转
- **伏笔回收**：之前埋下的伏笔如何回收
- **留白与想象**：给读者的思考空间

## 使用流程

1. 向用户询问小说类型、灵感碎片、预计篇幅、风格基调等基本信息。
2. 根据用户输入，按照小说大纲核心内容的模块，逐步输出内容，每一步完成后，询问用户是否需要调整修改。

## 输出格式

1. 按步骤输出每个模块的内容，每个模块完成后，询问用户是否需要调整修改。
2. 最终输出完整大纲，将讨论结果整理到项目的对应目录：
   - `01_World/world_setting.md`：世界观设定
   - `02_Characters/`：角色卡（每个主要角色单独文件）
   - `03_Outline/main_outline.md`：全书总纲，如果采用分卷规划，需要包含分卷结构配置，模板参考项目下的 `./volume_structure.md`，没有则参考插件内置的`/templates/volume_structure.md`
   - **分卷细纲规划输出**（如果进行了分卷规划）：
     - `03_Outline/volume_<卷号>_detailed.md`：各卷的细纲模板（卷级细纲），模板参考项目下的 `./volume_detailed.md`，没有则参考插件内置的`/templates/volume_detailed.md`
3. **版本控制**：重要的构思节点自动提交git记录

## 交互原则

1. **渐进式生成**：一步一步来，不一次性输出所有内容
2. **实时调整**：用户可以随时修改任何已完成的部分
3. **智能推荐**：在关键节点提供建议选项
4. **灵活适配**：根据用户的反馈调整后续内容
5. **保持连贯**：修改某一部分时，关联内容同步更新

## 注意事项

1. 如果用户提供的信息不完整，先用合理假设推进，后续可调整
2. 对于类型小说（如悬疑、恐怖），需要增加更多专业要素
3. 长篇和短篇的侧重点不同，注意篇幅平衡
4. 保持每个角色的动机合理，避免工具人
5. 情节设计要有起伏，不能平铺直叙

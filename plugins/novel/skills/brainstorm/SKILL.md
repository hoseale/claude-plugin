# 小说大纲生成器

逐步生成小说大纲，按模块创建，每个模块完成后可调整。

## 交互原则

1. **渐进式生成**：逐步输出
2. **实时调整**：随时修改
3. **智能推荐**：关键点提供建议
4. **灵活适配**：根据反馈调整
5. **保持连贯**：关联内容同步更新

### Step1：类型与卖点引擎

#### 1. 作品类型

- 修仙爽文
- 都市神豪
- 女频虐恋
- 无限流
- 悬疑推理

#### 2. 目标读者

- 男频 / 女频
- 年龄层
- 阅读偏好（爽/虐/智斗）

#### 3. 核心卖点（必须具体）

例如：

- 废柴逆袭 + 无限升级
- 被背叛后复仇
- 扮猪吃虎

#### 4. 爽点类型选择

至少3种：
打脸、逆袭、装逼、反杀、情绪爆发、智斗

#### 5. 爽点强度设定

弱 → 中 → 强 → 爆
决定节奏风格

### Step2：确定基本信息

- **作品标题**
- **作品类型**：言情/玄幻/悬疑等
- **风格基调**：轻松/沉重/黑暗等
- **预计篇幅**：短/中/长/超长篇
- **一句话简介**：[主角]+[困境]+[目标]+[阻碍]
- **一句话卖点**：[一句话卖点]
- **核心主题**：中心思想

### Step3：核心冲突与反派系统

核心冲突：

- 主角目标 vs 阻碍

三层反派结构

- 终极反派（贯穿全书）
- 阶段反派
- 小反派

核心转变（必须绑定主题）

### Step4：确定人物小传

- 基础信息：姓名、年龄、性别、外貌
- 背景
- 性格
- 核心动机
- 成长弧光
- 关系网
  - 反派
  - 配角
- 该角色能制造什么爽点？

### Step5：世界观设定

- 按剧情需要生成
- 优先服务冲突与爽点

#### 基础物理与地理环境

1. 世界名称
2. 地理布局
   - 地图概况：大陆形状、海域分布、特殊地貌
   - 区域划分：东西南北中各有什么国家/势力/禁地
   - 关键地标：主角出生地、主要副本地点、重要城市
3. 自然环境
   - 历法/时间： 一年几个月？一天几小时？有没有特殊的纪年法（如：龙历）
   - 天气： 有几个太阳？几个月亮？有没有特殊的天象
   - 物种生态：有哪些特殊动植物

#### 力量体系与等级设定

这是网文/幻想小说的核心，决定了“爽点”和“升级路径”。

1. **力量本质**：这种力量叫什么？（灵气、查克拉、异能、超能力、科技源力）？它是如何产生的？
2. **等级体系**
   - **等级划分**：人物如何提升等级？（如：完成任务、获得经验、与人互动等）
   - **晋升条件**：如何升级？
   - **能力差异**：每个等级之间的差距有多大？（高一阶是否能碾压低一阶？）
3. **战斗表现形式**:
   - **技能/术法的分类**：攻击、防御、辅助
   - **武器/装备的分类**：法宝、神器、机甲
4. **代价与限制**：使用力量是否有副作用？（如：折寿、走火入魔、消耗理智）。这一点非常重要，限制能带来戏剧冲突。

#### 社会势力与政治结构

这部分决定了主角与谁为敌，与谁结盟。

1. 势力分布图：
   - 顶级势力：帝国、宗门、联邦政府、巨型财团。
   - 中层势力：地方豪强、中型帮派、学院。
   - 底层势力：贫民窟帮派、小村落。
2. 政治体制：谁在统治世界？（皇帝、议会、长老团、AI统治）。
3. 势力关系：
   - 敌对阵营（正在打仗/冷战）。
   - 联盟阵营（利益交换）。
   - 中立阵营（隐世家族）。

#### 经济与资源体系

金钱是推动剧情的重要动力，避免货币体系崩溃。

1. 货币系统：
   - 货币名称和换算比例（如：1金币=100银币）。
   - 购买力参照（如：1金币等于普通家庭一个月生活费）。
2. 核心资源：
   - 大家在争夺什么？（矿石、石油、晶核、异兽内丹）。
   - 资源分布不均导致的矛盾（这也是剧情冲突的来源）。
3. 职业体系：
   - 除了战斗人员，普通人靠什么生活？（平民、炼药师、铁匠、驯兽师、向导）。

#### 历史背景与重大事件

故事的背景板，增加故事的厚重感。

1. 编年史：重要的历史节点。
2. 远古传说：流传下来的神话或谣言（通常涉及结局的秘密）。
3. 近期大事件：故事开始前10年-100年内发生的大事，直接影响了主角当下的处境（如：十年前的战争导致男主成为孤儿）。

#### 文化与禁忌

增加代入感，让世界“活”起来。

1. 宗教信仰：人们信奉什么神？有什么仪式？
2. 风俗习惯：婚丧嫁娶的习俗、特殊的节日庆典。
3. 世界禁忌：在这个世界绝对不能做的事是什么？（如：不能直视神像、不能提及某个名字）。

### Step6：确定情节结构

- 起（钩子强）
- 承（连续爽点）
- 转（压制）
- 合（爆发）

### Step7：爽点骨架

全书爽点规划

- 阶段1：弱爽（试探）
- 阶段2：中爽（成长）
- 阶段3：强爽（碾压）
- 阶段4：爆爽（终局）

爽点节奏规则

- 每3章：小爽点
- 每10章：中高潮
- 每卷：大高潮

标记关键爽点事件

- 第一个打脸：
- 第一次反杀：
- 第一次大爆发：

### Step8：结局设计

- **结局类型**：圆满/悲剧/开放/反转
- **伏笔回收**：之前埋下的伏笔如何回收
- **留白与想象**：给读者的思考空间

### Step9：文风与语感定制

1. 根据题材为写手制定《文风指南》
2. 提炼题材专属**名词库**
3. 提炼**禁用词汇**
4. 规则插入到`style_guide.md`最后

### Step10：输出文档

- `01_World/world_setting.md`：世界观
- `02_Characters/`：角色卡（每个角色单独文件）
- `03_Outline/main_outline.md`：大纲内容（不含世界观、角色卡）

### Step11：调用skill(volume-outline)

- 输入：世界观、人物、主线冲突、爽点骨架

## 版本控制

git commit -m "{概要}"

## 注意事项

- 类型优先于世界观
- 爽点必须前置设计
- 不允许“慢热无钩子”
- 主角必须有成长弧光
- 所有设定服务于“读者爽感”

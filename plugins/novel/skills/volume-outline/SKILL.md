# 概述

根据小说大纲、世界观设定、人物设定、爽点骨架制定分卷大纲，也可根据用户需求修改分卷大纲。

### Step1：边界定义

**卷名设定**：四字以内（必须具备传播力）

**地图圈定**：

1. 活动范围（主地图）
2. 入场点（如何进入）
3. 离场点（为何离开）

**篇幅预估**：

- 建议范围：15万 - 25万字
- 必须在本卷内解决主要矛盾（禁止拖沓）

### Step4：爽点结构设计

本卷必须明确爽点类型与分布：

爽点类型选择（至少3种）

- 打脸（身份反转）
- 逆袭（弱 → 强）
- 装逼（实力碾压）
- 反杀（绝境翻盘）
- 情绪爆发（复仇/守护）
- 智斗（策略取胜）

爽点节奏分布（必须填写）

| 阶段 | 爽点数量 | 类型          |
| ---- | -------- | ------------- |
| 起   | 1-2      | 初始装逼/异常 |
| 承   | 4-6      | 连续成长+打脸 |
| 转   | 2-3      | 压制/低谷     |
| 合   | 2-3      | 大爆发        |

### Step5：剧情推演

采用 "四段式推演" 填充本卷骨架:

**起（入局）**

- 新地图
- 目标确立
- 初次冲突

**承（发展）**

- 小副本
- 获取资源
- 初次胜利 + 挫折

**转（危机）**

- 反派反扑
- 主角低谷
- 重大抉择

**合（高潮）**

- 决战
- 爽点集中爆发
- 收益兑现
- 引出下一卷线索

### Step6：连载节奏验证

必须检查：

- 每3章：一个小爽点
- 每10章：一个中高潮
- 卷末：超级高潮

情绪曲线:

```text
上升 → 挫折 → 爆发 → 再压制 → 最终爆发
```

## 审核流程

审核以下项目：

- 输出结果是否符合**执行流程**
- 世界观是否偏离或存在冲突
- 大纲`main_outline.md`是否偏离或存在冲突
- 角色设定是否偏离或存在冲突
- 时间线矛盾排查：人物行踪是否有分身术？时间流逝是否合理？
- 战力/规则冲突排查：是否出现战力崩坏或与设定矛盾的能力？
- 人设OOC排查：角色的决策是否违背了其核心动机？

## 交互原则

1. **渐进式生成**：逐步完成每个步骤，用户可中途调整。
2. **实时验证**：每一步完成后验证合理性，确保逻辑连贯。
3. **模板化输出**：严格按照分卷大纲表模板生成最终文档。
4. **上下文感知**：基于已有的大纲和世界观设定进行创作，保持一致性。
5. **灵活调整**：根据用户反馈调整分卷细节，确保满足创作需求。

## 文件输出位置

## 版本控制

git commit -m "{简介}"

## 注意事项

1. **闭环设计**：确保每卷有完整的起承转合，避免剧情拖沓。
2. **成长合理**：人物状态变迁应有合理幅度，避免开挂或停滞。
3. **地图清晰**：活动范围明确，避免地图混乱。
4. **节奏控制**：根据篇幅预估合理分配剧情节点。

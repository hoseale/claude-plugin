# 概述

根据小说大纲、世界观设定、人物设定，制定分卷大纲，也可根据用户需求修改分卷大纲。

## 执行流程

**禁止合并步骤执行**，先列出todos，执行完成标记完成。

### 步骤一：边界定义

**卷名设定**：用四个字以内概括本卷主题（如：《初露锋芒》、《兽潮觉醒》、《王都风云》）。

**地图圈定**：

1. 确定本卷的主要活动范围。
2. 标记"入场点"（如何来到这里）和"离场点"（为何离开去下一张地图）。

**篇幅预估**：设定本卷字数上限（如：20万字），强制在此范围内解决主要矛盾，防止注水。

### 步骤二：状态锚定

**起始状态**：

- **实力**：等级、装备、持有资金。
- **心态**：当前的迫切需求

**终结状态**：

- **实力**：本卷结束时的等级/身份变化。
- **心态**：人物的核心欲望是否发生了转移？（如：从“活下去”变成了“想要守护团队”）

**变化差值**：对比起止状态，确认本卷的成长幅度是否合理。

### 步骤三：本卷核心冲突

**主要矛盾**：主角在本卷最大的阻碍是什么？

**本卷反派**：谁制造了这个阻碍？（设定反派的结局：被打败、被打脸、被转化）。

**高潮事件**：本卷最后的大决战/大事件是什么？（需具备画面感，是全卷情绪的最高点）。

### 步骤四：剧情推演

采用 "四段式推演" 填充本卷骨架：

**起（开端）**：新地图开启，主角确立本卷小目标。

**承（发展）**：遭遇小怪，结识盟友，获取资源，第一次小挫折。

**转（危机）**：反派反扑，主角陷入绝境/低谷，必须做出艰难抉择。

**合（高潮与收尾）**：绝地反击，解决反派，收获战利品，引出下一卷线索。

### 步骤五：卖点与情绪验证

**核心看点**：本卷主要"卖"什么？（如：打脸爽文、智斗解谜、虐心恋情）。

## 输出模板：分卷大纲表

```
【第 X 卷：[卷名]】

一、基础定位

预估字数：[如：20万字]
主要地图：[本卷发生地 / 入场方式 / 离场方式]
核心卖点：[本卷主要卖什么？爽文/智斗/虐恋]

二、人物成长锚点

起始状态：等级[ ] / 身份[ ] / 核心诉求[ ]
终结状态：等级[ ] / 身份[ ] / 核心诉求[ ]
人员变动：[新增伙伴 / 离开或死亡的角色]

三、核心冲突

核心矛盾：[主角想要什么 vs 谁在阻碍]
本卷反派/BOSS：[具体是谁 / 结局如何]
高潮名场面：[本卷结尾最燃/最催泪的一幕]

四、剧情推演

起（入局）：[开篇事件，确立目标]
承（试炼）：[小挫折、小胜利、获取关键道具]
转（危机）：[至暗时刻，最大的阻碍显现]
合（决战）：[解决最终BOSS，收获战利品]

五、伏笔与衔接（对应步骤一、四）

本卷伏笔：[埋下的雷或糖]
下卷钩子：[为什么要开启下一张地图？]

```

## 交互原则

1. **渐进式生成**：逐步完成每个步骤，用户可中途调整。
2. **实时验证**：每一步完成后验证合理性，确保逻辑连贯。
3. **模板化输出**：严格按照分卷大纲表模板生成最终文档。
4. **上下文感知**：基于已有的大纲和世界观设定进行创作，保持一致性。
5. **灵活调整**：根据用户反馈调整分卷细节，确保满足创作需求。

## 文件输出位置

所有分卷大纲整合到`03_Outline/volume_structure.md`文件

## 版本控制

git commit -m "{简介}"

## 注意事项

1. **闭环设计**：确保每卷有完整的起承转合，避免剧情拖沓。
2. **成长合理**：人物状态变迁应有合理幅度，避免开挂或停滞。
3. **地图清晰**：活动范围明确，避免地图混乱。
4. **节奏控制**：根据篇幅预估合理分配剧情节点。

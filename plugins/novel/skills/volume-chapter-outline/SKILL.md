# 概述

将分卷大纲拆解为具体章节序列，确立每一章的"功能"与"钩子"，确保节奏紧凑。

## 执行流程

**禁止合并步骤执行**，先列出todos，执行完成标记完成。

### 步骤一：章节预算分配

**输入**：

- 确定是第几卷，以及本卷预估总字数（如：20万字）
- 读取大纲、分卷大纲、世界观设定、角色设定

**计算**：按照每章3000-4000字计算，得出总章节数

**比例切分**：将总章节数按"四幕结构"分配：

| 阶段       | 占比 | 约章节数 | 内容定位                             |
| ---------- | ---- | -------- | ------------------------------------ |
| 起（铺垫） | 15%  | 约12章   | 入场、立目标、展现实力               |
| 承（发展） | 50%  | 约40章   | 小副本、打怪升级、获取资源、遭遇挫折 |
| 转（危机） | 20%  | 约16章   | 陷害、低谷、反转、大战前夕           |
| 合（高潮） | 15%  | 约12章   | 决战、收获、清算、引出下卷           |

### 步骤三：逐章节点设计

为每一章赋予具体的任务。必须包含三个要素：

- **核心事件**：这章发生什么？
- **功能标签**：铺垫/爽点/转折/伏笔/虐点
- **钩子**：悬念是什么？（钩子可在章尾或中间，吸引读者点开下一章）

### 步骤四：节奏校验

| 校验项   | 检查标准                         | 处理方法               |
| -------- | -------------------------------- | ---------------------- |
| 爽点密度 | 检查是否连续3章以上没有爽点/进展 | 插入冲突或小高潮       |
| 信息密度 | 检查是否连续多章全是说明文       | 将说明融入对话或冲突中 |

## 交互原则

1. **渐进式生成**：逐步完成每个步骤，用户可中途调整
2. **黄金三章**：前3章必须重点设计，确保"冲突前置"、"金手指出现"、"第一个小爽点"都在前3章完成
3. **钩子思维**：[钩子]栏必须填写，如果写不出钩子说明这章太平淡，可以考虑删掉或合并
4. **粒度控制**：
   - 快节奏爽文：细化到每一章
   - 慢热正剧：先细化到"事件组"，写的时候再临时发挥

## 注意事项

1. **闭环设计**：确保每卷有完整的起承转合
2. **成长合理**：人物状态变迁应有合理幅度
3. **节奏紧凑**：避免注水，每章都要有进展
4. **钩子连贯**：章尾钩子要诱人，让读者忍不住点开下一章

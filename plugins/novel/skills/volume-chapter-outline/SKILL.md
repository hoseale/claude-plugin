# 概述

将分卷大纲拆解为具体章节序列，确立每一章的"功能"与"钩子"，确保节奏紧凑。

## 执行流程

**禁止合并步骤执行**，先列出todos，执行完成标记完成。

- [ ] Step1：章节预算分配

- [ ] Step5：断章与钩子强化
- [ ] Step6：节奏校验
- [ ] Step7：输出章节表

### Step1：章节预算分配

**输入**：

- 确定是第几卷，以及本卷预估总字数（如：20万字）
- 读取大纲、分卷大纲、世界观设定、角色设定

**计算**：按照每章3000-4000字计算，得出总章节数

**比例切分**：将总章节数按"四幕结构"分配：

| 阶段       | 占比 | 内容定位       |
| ---------- | ---- | -------------- |
| 起（铺垫） | 15%  | 快速抓住读者   |
| 承（发展） | 50%  | 连续爽点       |
| 转（危机） | 20%  | 压制           |
| 合（高潮） | 15%  | 爆发、引出下卷 |

### Step3：爽点分配

爽点类型选择

- 打脸（身份反转）
- 逆袭（弱 → 强）
- 装逼（实力碾压）
- 反杀（绝境翻盘）
- 情绪爆发（复仇/守护）
- 智斗（策略取胜）

爽点章节分布：
| 爽点类型 | 分布 |
| ---- | ----- |

爽点章节分布规则：

- 每3章必须有爽点
- 不允许连续5章无进展

### Step4：逐章设计

每章必须包含：

- 核心事件：
- 功能标签：（爽点/铺垫/反转/伏笔/虐点）
- 爽点类型：（如有）
- 信息推进：（世界/人物/主线）
- 钩子：（读完整章的动力）
- 断章方式：(合适的断章位置, 让读者想读下一章)

黄金三章：

- 第1章：
  - 冲突出现
  - 世界异常
  - 钩子：重大疑问
- 第2章：
  - 金手指/外挂/异常强化
- 第3章：
  - 第一个爽点（必须）

### Step5：断章与钩子强化

钩子类型：

- 信息差（他不知道真相）
- 危机（马上要死）
- 反转（其实是陷阱）
- 新目标（更大事件出现）

断章规则：

- 80%章节必须断在“未完成事件”
- 禁止“平稳结尾”

### Step6：节奏校验

| 校验项   | 检查标准                         |
| -------- | -------------------------------- |
| 爽点密度 | 检查是否连续3章以上没有爽点/进展 |
| 信息推进 | 每章必须推进                     |
| 钩子强度 | 读完整章的动力                   |
| 断章方式 | 读者想读下一章                   |

## 交互原则

1. **渐进式生成**：逐步完成每个步骤，用户可中途调整。
2. **实时验证**：每一步完成后验证合理性，确保逻辑连贯。
3. **模板化输出**：严格按照分卷大纲表模板生成最终文档。
4. **上下文感知**：基于已有的大纲和世界观设定进行创作，保持一致性。
5. **灵活调整**：根据用户反馈调整分卷细节，确保满足创作需求。

## 注意事项

1. **闭环设计**：确保每卷有完整的起承转合
2. **成长合理**：人物状态变迁应有合理幅度
3. **节奏紧凑**：避免注水，每章都要有进展
4. **钩子连贯**：章尾钩子要诱人，让读者忍不住点开下一章

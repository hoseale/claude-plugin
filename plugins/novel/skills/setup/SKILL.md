# 技能：setup-novel

你现在的任务是初始化一个结构化的小说项目环境。请按照以下步骤严格执行：

**验证当前环境**：检查当前目录是否已经存在小说项目结构（是否有 `01_World/` 等目录）。如果已存在，询问用户是否要重新初始化（注意：重新初始化可能覆盖现有文件）。

1. 创建相应的文件夹结构：

- `00_Brainstorm` (用于存放杂记和灵感)
- `01_World` (世界观与设定)
- `02_Characters` (角色卡)
- `03_Outline` (大纲与前情提要)
- `04_Chapters/` (小说正文目录，内部按卷组织)

2. 执行通用初始化步骤（无论是否启用分卷模式）：
   - 在 `01_World` 中创建 `world_setting.md`。
   - 在 `03_Outline` 中创建 `main_outline.md`。
   - 复制插件根目录下的 `/templates/volume_structure.md` 到项目的根目录。
   - 复制插件根目录下的 `/templates/volume_detailed.md` 到项目的根目录。
   - 复制插件根目录下的 `/templates/style_guide.md` 到项目的根目录。

3. 在终端运行 `git init`（如果尚未初始化），并进行首次 commit "Initialize Novel Project"。

4. 告诉用户初始化已完成，并根据分卷模式提供相应提示：
   - 提示用户可以通过 `/brainstorm` agent 进行分卷规划，然后使用 `/write-chapter` 开始写作。

## 错误处理

- 如果目录创建失败（权限问题），通知用户并停止执行。
- 如果文件写入失败，尝试使用备用方案（如创建空文件）。
- 如果git已经初始化，跳过`git init`步骤但仍尝试提交初始文件。
- 任何步骤失败时，清理已创建的资源或通知用户手动清理。

## 分卷模式特有错误处理

- 如果分卷模板文件不存在但用户选择了分卷模式，创建基本的分卷结构配置文件。
- 如果卷目录创建失败，回退到兼容模式并通知用户。
- 如果分卷配置文件写入失败，继续其他初始化步骤但警告用户需要手动配置分卷。

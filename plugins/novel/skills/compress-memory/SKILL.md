# 技能：compress-memory

你的任务是为小说提取前情提要，以防止后续写作时上下文窗口超载。

## 执行流程：

**验证项目结构**：检查 `03_Outline/` 和 `04_Chapters/` 目录是否存在。如果 `03_Outline/summary.md` 不存在，创建该文件。

**检测分卷模式**：检查 `03_Outline/volume_<卷号>_detailed.md` 是否存在。

- 如果存在，进入分卷模式
- 如果不存在，进入兼容模式

1. **扫描章节**：
   - **分卷模式**：扫描 `04_Chapters/volume_*/` 下的所有章节，按卷和章节号排序
   - **兼容模式**：扫描 `04_Chapters/` 下的所有章节，按章节号排序
2. **确定未总结章节**：对比 `03_Outline/summary.md` 中的已有记录，找出尚未被总结的章节。
   - 如果summary.md为空，总结所有章节
   - 如果已有记录，找出最后被总结的章节，从该章节之后开始总结
   - **分卷模式**：按卷处理，确保各卷章节被正确识别
3. 仔细阅读这些未总结的章节文本。
4. 提炼出核心摘要：
   - 关键剧情推进。
   - 角色关系的实质性变化。
   - 埋下的伏笔或新增的悬念。
   - 重要道具/能力的获得与损失。
5. **保存摘要**：将提取出的摘要追加到 `03_Outline/summary.md` 文件末尾。
   - **分卷模式**：按卷组织摘要，添加卷标题（如"## 第一卷摘要"），然后列出该卷章节摘要
   - **兼容模式**：直接按章节号顺序追加摘要
   - 每个章节摘要应包含章节号和简要标题（如果有）
6. **报告结果**：向用户报告压缩结果。
   - 列出被总结的章节（分卷模式时注明卷号）
   - 估计释放的阅读负担（字数、token数估算）
   - 提示用户现在可以在写作时参考summary.md而非阅读完整章节

## 错误处理

- 如果 `04_Chapters/` 目录为空或不存在，通知用户并停止执行。
- 如果文件读取失败（权限或编码问题），跳过该章节并记录警告。
- 如果 `summary.md` 写入失败，尝试创建备份或通知用户手动保存摘要。
- 遇到无法解析的章节编号时，按文件修改时间排序处理。

## 分卷模式特有处理

- 如果卷目录不存在但分卷配置文件存在，按兼容模式处理并警告用户
- 如果分卷模式下的章节文件路径不符合预期，尝试自动修正
- 处理跨卷章节引用时保持正确性

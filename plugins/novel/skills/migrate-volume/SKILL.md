# 技能：migrate-volume

你的任务是将现有小说项目从兼容模式迁移到分卷模式。这个过程会将章节文件按卷组织，并创建分卷配置文件。

## 迁移前准备

1. **项目验证**：检查当前目录是否为有效的小说项目（包含 `01_World/`, `02_Characters/`, `03_Outline/`, `04_Chapters/` 目录）。
2. **备份警告**：提醒用户迁移操作会移动章节文件，建议先备份项目。
3. **分卷检测**：检查是否已启用分卷模式（检查 `03_Outline/volume_structure.md` 是否存在）。如果已启用，询问用户是否要重新规划分卷。

## 迁移流程

### 1. 分析现有章节
- 扫描 `04_Chapters/` 目录下的所有章节文件
- 按修改时间或文件名排序章节
- 统计总章节数，分析章节内容（可选）

### 2. 设计分卷方案
- 根据章节数量建议分卷数：
  - 20章以下：建议1卷
  - 20-50章：建议2-3卷
  - 50-100章：建议3-5卷
  - 100章以上：建议5-8卷
- 与用户讨论分卷点，考虑：
  - 情节转折点
  - 时间跳跃
  - 地点变更
  - 主题转换
- 确定每卷的章节范围和卷名

### 3. 创建分卷结构
- 在 `03_Outline/` 中创建 `volume_structure.md`，基于模板填充分卷信息
- 为每卷创建细纲模板文件（如 `volume_1_detailed.md`）
- 在 `04_Chapters/` 下创建卷目录（如 `volume_1/`, `volume_2/`）

### 4. 移动章节文件
- 根据分卷方案将章节文件移动到对应卷目录
- 保持文件命名一致性：`ch<两位数字>.md`
- 更新章节内的交叉引用（如果有）

### 5. 提取细纲内容
- 从现有章节内容中提取关键信息填充细纲
- 包括：主要场景、出场人物、情节要点、伏笔设置
- 保持细纲的简洁性，避免过度详细

### 6. 验证迁移结果
- 检查所有章节文件是否已正确移动
- 验证分卷配置文件是否正确
- 测试 `write-chapter` 技能是否能在分卷模式下正常工作

## 输出文件

迁移完成后将创建以下文件：

1. `03_Outline/volume_structure.md` - 分卷结构配置
2. `03_Outline/volume_<卷号>_detailed.md` - 各卷细纲（基于章节内容生成）
3. `04_Chapters/volume_<卷号>/` - 卷目录，包含移动后的章节文件

## 错误处理

1. **文件冲突**：如果目标文件已存在，询问用户如何处理（覆盖、跳过、重命名）
2. **目录权限**：如果没有创建目录的权限，尝试在 `04_Chapters/` 下直接使用带卷号前缀的文件名
3. **章节丢失**：如果迁移后章节文件丢失，提供回滚方案
4. **内容损坏**：确保移动文件时内容不被损坏
5. **git历史**：保留git历史记录，使用git移动文件而非删除/创建

## 用户确认点

在整个迁移过程中，需要在以下关键点获取用户确认：

1. **开始迁移前**：确认用户已备份项目
2. **分卷方案**：确认用户接受建议的分卷方案
3. **文件移动前**：显示将移动的文件列表供用户确认
4. **完成迁移后**：显示迁移摘要供用户验证

## 迁移后提示

迁移完成后，提醒用户：

1. 分卷模式已启用，后续写作将使用分卷结构
2. 可以使用 `/brainstorm` agent 进一步细化分卷规划
3. `write-chapter` 技能现在会询问卷号
4. 可以使用 `compress-memory` 技能按卷生成摘要
5. 如果需要调整分卷，可以手动编辑 `volume_structure.md`

# 技能：migrate-volume

将小说项目从兼容模式迁移到分卷模式，按卷组织章节文件。

## 迁移前准备

1. **项目验证**：检查是否为有效小说项目
2. **备份警告**：提醒备份，迁移会移动文件
3. **分卷检测**：检查是否已启用分卷模式，如已启用询问是否重新规划

## 迁移流程

### 1. 分析现有章节
- 扫描 `04_Chapters/` 下的章节文件
- 按修改时间或文件名排序
- 统计章节数，分析内容（可选）

### 2. 设计分卷方案
- 根据章节数建议分卷数
- 与用户讨论分卷点，考虑：
  - 情节转折
  - 时间跳跃
  - 地点变更
  - 主题转换
- 确定每卷章节范围和卷名

### 3. 创建分卷结构
- 在 `03_Outline/` 创建 `volume_structure.md`
- 为每卷创建细纲模板文件
- 在 `04_Chapters/` 下创建卷目录

### 4. 移动章节文件
- 按分卷方案移动文件到对应卷目录
- 保持命名一致性：`ch<两位数字>.md`
- 更新交叉引用（如有）

### 5. 提取细纲内容
- 从章节内容提取关键信息填充细纲
- 包括：场景、人物、情节、伏笔
- 保持简洁，避免过度详细

### 6. 验证迁移结果
- 检查章节文件是否正确移动
- 验证分卷配置文件
- 测试 `write-chapter` 在分卷模式下工作

## 输出文件

迁移完成后将创建以下文件：

1. `03_Outline/volume_structure.md` - 分卷结构
2. `03_Outline/volume_<卷号>_detailed.md` - 卷细纲
3. `04_Chapters/volume_<卷号>/` - 卷目录

## 错误处理

1. **文件冲突**：目标文件已存在，询问处理方式
2. **目录权限**：无权限时尝试带卷号前缀文件名
3. **章节丢失**：提供回滚方案
4. **内容损坏**：确保移动时内容不损坏
5. **git历史**：保留历史，使用git移动而非删除/创建

## 用户确认点

在整个迁移过程中，需要在以下关键点获取用户确认：

1. **开始迁移前**：确认已备份
2. **分卷方案**：确认接受方案
3. **文件移动前**：显示移动文件列表
4. **完成迁移后**：显示迁移摘要

## 迁移后提示

迁移完成后，提醒用户：

1. 分卷模式已启用
2. 可用 `/brainstorm` 细化规划
3. `write-chapter` 会询问卷号
4. 可用 `compress-memory` 按卷生成摘要
5. 可手动编辑 `volume_structure.md` 调整分卷
